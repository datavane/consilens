# 配置详解

Consilens 当前使用结构化的 CLI 配置模型，配置入口对应 *CliConfiguration.java*。

## 完整示例

```yaml
source:
  type: mysql
  url: jdbc:mysql://localhost:3306/source_db?useSSL=false&serverTimezone=UTC
  username: root
  password: secret

target:
  type: postgresql
  url: jdbc:postgresql://localhost:5432/target_db?currentSchema=public&ssl=false&ApplicationName=consilens
  username: postgres
  password: secret

comparison:
  tables:
    source: public.orders
    target: public.orders
  keys:
    source:
      - order_id
    target:
      - order_id
  compareColumns:
    source:
      - order_id
      - customer_id
      - amount
      - status
      - created_at
    target:
      - order_id
      - customer_id
      - amount
      - status
      - created_at
  where:
    source: created_at >= '2026-01-01'
    target: created_at >= '2026-01-01'

strategy:
  mode: checksum
  algorithm: xor
  bisectionFactor: 8
  bisectionThreshold: 5000
  batchSize: 1000
  enableProfiling: true
  localCompare:
    mode: full

result:
  sinks:
    - format: console
      type: result
    - format: json
      type: diff-record
      properties:
        path: ./results/orders_diff.json
        pretty: true
    - format: table
      type: diff-record
      properties:
        url: jdbc:mysql://localhost:3306/consilens_results
        username: writer
        password: secret
        prefix: diff_results_
        suffixTimestamp: true
        createTable: true
        batchSize: 1000

normalization:
  global:
    decimal:
      precision: 2
      rounding: true
    timestamp:
      format: yyyy-MM-dd HH:mm:ss
      timezone: UTC
```

## 一、连接与表配置

### 数据库连接

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `source.url` | 是 | 源库 JDBC URL |
| `source.username` | 是 | 源库用户名 |
| `source.password` | 否 | 源库密码 |
| `source.type` | 否 | 显式数据库类型，未填时从 URL 推断 |
| `target.url` | 是 | 目标库 JDBC URL |
| `target.username` | 是 | 目标库用户名 |
| `target.password` | 否 | 目标库密码 |
| `target.type` | 否 | 显式数据库类型 |

### 表与列

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `comparison.tables.source` | 是 | 源表名，支持 `schema.table` |
| `comparison.tables.target` | 是 | 目标表名 |
| `comparison.keys.source` | 是 | 源表主键列列表 |
| `comparison.keys.target` | 是 | 目标表主键列列表 |
| `comparison.compareColumns.source` | 否 | 源表参与比较列 |
| `comparison.compareColumns.target` | 否 | 目标表参与比较列 |
| `comparison.where.source` | 否 | 源表过滤条件 |
| `comparison.where.target` | 否 | 目标表过滤条件 |

注意：

- 两侧主键列数量必须一致。
- `comparison.compareColumns.source` 和 `comparison.compareColumns.target` 若为空，当前代码不会自动补齐全表列，建议明确配置。
- 过滤条件会直接拼接到分段查询里，应保证两侧语义等价。

## 二、策略与算法

### `strategy`

支持值：

- `checksum`
- `join`
- `local`

实际状态：

- `checksum`：已实现，支持跨库
- `join`：已实现，但要求两张表在同一个 JDBC URL
- `local`：枚举已定义，但 CLI 当前会直接抛出未实现异常

### `algorithm`

支持值：

- `concat`
- `xor`

建议：

- 小表或验证场景：`concat`
- 大多数大表：`xor`

说明：

- `algorithm` 只决定分段阶段的摘要计算方式
- `concat` 更直观，但通常更适合小表或验证场景
- `xor` 是跨库大表的默认推荐值

### 终局本地比较

终局本地比较默认使用完整行比较：

```yaml
strategy:
  localCompare:
    mode: full
```

可选值：

- `full`：默认值，直接拉取当前小段完整行做本地比较
- `row-hash`：先拉主键 + 行哈希，在本地找出差异键后再回查完整行

`row-hash` 只有显式配置时才会启用：

```yaml
strategy:
  localCompare:
    mode: row-hash
```

一般建议保持默认 `full`。只有在表非常宽、最终小段仍较大且差异行比例很低时，再考虑显式启用 `row-hash`。

## 三、checksum 递归参数

### `bisectionFactor`

首轮分段使用的目标分段数。

示例：

- `bisectionFactor: 8` 表示初始更偏向按 8 段切分

### `bisectionThreshold`

本地比较阈值，也是递归切换策略的核心参数。

当前实现行为：

1. 全表或大段先按 `bisectionFactor` 做首轮分段
2. 若子段 `maxRows > bisectionThreshold * bisectionFactor`，仍允许继续多路分段
3. 若子段 `maxRows <= bisectionThreshold * bisectionFactor`，切换为真正二分
4. 若当前段 `totalRows < bisectionThreshold`，直接本地比对

所以：

- “切到二分”的隐含阈值 = `bisectionThreshold * bisectionFactor`
- “切到本地拉数”的阈值 = `bisectionThreshold`

### `batchSize`

当前主要用于默认计算 `bisectionThreshold`：

- 如果未显式配置 `bisectionThreshold`
- 则默认取 `batchSize * 10`

### `enableProfiling`

启用后会增强过程信息记录，便于诊断性能问题。

## 四、并发配置

示例：

```yaml
concurrency:
  io:
    core: 16
    max: 64
    queueSize: 10000
    keepAliveSeconds: 60
    threadNamePrefix: consilens-io-
  cpu:
    core: 8
    max: 16
    queueSize: 10000
    keepAliveSeconds: 60
    threadNamePrefix: consilens-cpu-
```

适用建议：

- IO 密集型场景：提高 `io.max`
- CPU 紧张机器：控制 `cpu.max`

## 五、输出配置（result.sinks）

输出通过 `result.sinks[]` 列表配置，每个条目对应一个 sink，可同时启用多个。

### 顶层字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `result.sinks` | 列表 | `[]` | sink 列表，顺序无关 |
| `result.parallelism` | 整数 | `4` | 保留字段，暂未启用 |
| `result.failOnSinkError` | 布尔 | `false` | sink 失败时是否中断主流程 |

每个 sink 的公共字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `format` | 是 | 输出格式：`table` / `json` / `csv` / `console` |
| `type` | 是 | 数据类型：`diff-record`（差异明细）/ `result`（最终汇总） |
| `enabled` | 否 | 是否启用，默认 `true` |
| `properties` | 否 | 各 format 的专属扩展属性，YAML 嵌套对象 |

### console format

打印到标准输出，适合调试和本地运行。

```yaml
result:
  sinks:
    - format: console
      type: result
      properties:
        maxRows: 100
        showStatistics: true
```

`type: result` 时打印汇总统计；`type: diff-record` 时打印部分差异明细行（最多 `maxRows` 行）。

`console` 专属属性：

| 属性 | 默认值 | 说明 |
| --- | --- | --- |
| `maxRows` | `100` | 最多打印的差异明细行数，`-1` 表示不限制 |
| `showStatistics` | `true` | 是否打印统计汇总 |

### json format

将结果写出为 JSON 文件。

```yaml
result:
  sinks:
    - format: json
      type: diff-record
      properties:
        path: ./output/diff-${taskId}.json
        pretty: true

    - format: json
      type: result
      properties:
        path: ./output/summary-${taskId}.json
        pretty: false
```

`json` 专属属性：

| 属性 | 默认值 | 说明 |
| --- | --- | --- |
| `path` | 无 | 输出文件路径，支持 `${变量名}` 占位符 |
| `pretty` | `false` | 是否格式化输出（缩进） |
| `columns` | `[]` | 自定义字段映射列表（见下文），为空时序列化原始对象 |

**自定义字段示例（json diff-record）：**

```yaml
- format: json
  type: diff-record
  properties:
    path: ./output/diff-${taskId}.json
    pretty: true
    columns:
      - name: taskId
        value: ${taskId}
      - name: operation
        value: ${operation}
      - name: primaryKey
        value: ${primaryKey}
      - name: srcAmount
        value: ${src.amount}
        defaultValue: "0"
      - name: tgtAmount
        value: ${tgt.amount}
        defaultValue: "0"
      - name: env
        value: production
```

### csv format

将差异明细写出为 CSV 文件。

```yaml
result:
  sinks:
    - format: csv
      type: diff-record
      properties:
        path: ./output/diff-${taskId}.csv
        delimiter: ","
        includeHeader: true
        columns:
          - operation
          - primaryKey
          - sourceValues
          - targetValues
```

`csv` 专属属性：

| 属性 | 默认值 | 说明 |
| --- | --- | --- |
| `path` | 无 | 输出文件路径，支持 `${变量名}` 占位符 |
| `delimiter` | `,` | 列分隔符 |
| `includeHeader` | `true` | 是否输出表头行 |
| `columns` | `[]` | 自定义字段映射列表（见下文），为空时输出默认 4 列 |
| `mergeDefaults` | `false` | 为 `true` 时保留默认列并用 `columns` 覆盖或追加 |

**自定义字段示例（csv diff-record）：**

```yaml
- format: csv
  type: diff-record
  properties:
    path: ./output/diff-${taskId}.csv
    columns:
      - name: task_id
        value: ${taskId}
      - name: op
        value: ${operation}
      - name: pk
        value: ${primaryKey}
      - name: src_amount
        value: ${src.amount}
        defaultValue: "0"
      - name: tgt_amount
        value: ${tgt.amount}
        defaultValue: "0"
      - name: env
        value: production
```

### table format

将差异明细或最终汇总写入数据库表。

```yaml
result:
  sinks:
    - format: table
      type: diff-record
      properties:
        url: jdbc:mysql://localhost:3306/consilens_results
        username: writer
        password: ${RESULT_DB_PASSWORD}
        driver: com.mysql.cj.jdbc.Driver
        maxPoolSize: 10
        prefix: diff_result_
        suffixTimestamp: true
        createTable: true
        dropIfExists: false
        defaultColumnLength: 500
        batchSize: 1000
```

`table` 专属属性：

| 属性 | 默认值 | 说明 |
| --- | --- | --- |
| `url` | 无 | JDBC URL |
| `username` | 无 | 数据库用户名 |
| `password` | 无 | 数据库密码 |
| `driver` | 自动推断 | JDBC 驱动类名 |
| `maxPoolSize` | `5` | 连接池最大连接数 |
| `tableName` | 无 | 显式表名；设置后忽略 `prefix` 和 `suffixTimestamp` |
| `prefix` | `diff_result_` | 表名前缀 |
| `suffixTimestamp` | `true` | 是否在表名后追加时间戳（`yyyyMMdd_HHmmss`） |
| `createTable` | `true` | 是否自动建表 |
| `dropIfExists` | `false` | 建表前是否先 DROP TABLE |
| `defaultColumnLength` | `255` | VARCHAR 列默认长度 |
| `batchSize` | `1000` | 批量写入行数 |
| `columns` | `[]` | 自定义字段映射列表（见下文），为空时使用默认宽表模式 |

**自定义字段示例（table diff-record）：**

```yaml
- format: table
  type: diff-record
  properties:
    url: jdbc:mysql://localhost:3306/mydb
    username: root
    password: secret
    tableName: my_diff_detail
    createTable: true
    columns:
      - name: task_id
        value: ${taskId}
      - name: diff_type
        value: ${operation}
      - name: pk_value
        value: ${primaryKey}
      - name: src_amount
        value: ${src.amount}
        defaultValue: "0"
        columnType: DECIMAL(18,2)
      - name: tgt_amount
        value: ${tgt.amount}
        defaultValue: "0"
        columnType: DECIMAL(18,2)
      - name: env
        value: production
      - name: created_at
        value: ${timestamp}
        columnType: TIMESTAMP
```

### fields 字段映射说明

`columns` 适用于 `table`、`csv`、`json` 三种 format。每个字段由以下属性组成：

| 属性 | 必填 | 说明 |
| --- | --- | --- |
| `name` | 是 | 输出列/字段名称 |
| `value` | 否 | 值模板，支持 `${变量名}` 占位符和纯静态字符串混合 |
| `defaultValue` | 否 | 兜底值，当 `value` 解析结果为空时使用 |
| `columnType` | 否 | SQL 列类型（仅 `table` format 建表 DDL 使用），未设置时默认 `TEXT` |

**内置变量（diff-record 类型）：**

| 变量 | 说明 |
| --- | --- |
| `${taskId}` | 任务 ID |
| `${sourceTable}` | 源表名 |
| `${targetTable}` | 目标表名 |
| `${strategy}` | 比对策略 |
| `${algorithm}` | 比对算法 |
| `${operation}` | 差异操作类型（SOURCE_MISSING / TARGET_MISSING / MISMATCH） |
| `${primaryKey}` | 主键字符串 |
| `${changedColumns}` | 变化列 JSON 数组（源端） |
| `${changedColumns1}` | 源端变化列 JSON 数组 |
| `${changedColumns2}` | 目标端变化列 JSON 数组 |
| `${src.列名}` | 源端指定列的值 |
| `${tgt.列名}` | 目标端指定列的值 |
| `${timestamp}` | 当前时间 ISO-8601 字符串 |
| `${attr.key}` | DiffContext 自定义属性 |

**内置变量（result 类型，除上述公共变量外）：**

| 变量 | 说明 |
| --- | --- |
| `${status}` | 执行状态（EQUAL / DIFF） |
| `${totalDifferences}` | 差异总数 |
| `${sourceMissingCount}` | 源端缺失行数 |
| `${targetMissingCount}` | 目标端缺失行数 |
| `${mismatchCount}` | 不一致行数 |
| `${sourceRowCount}` | 源端总行数 |
| `${targetRowCount}` | 目标端总行数 |
| `${statistics_json}` | 完整统计 JSON：`{"totalCount":N,"mismatchCount":N,"sourceMissingCount":N,"targetMissingCount":N,"totalDiffCount":N,"accuracyRate":N}`，其中 `accuracyRate = (totalCount - totalDiffCount) / totalCount * 100`，保留 2 位小数（可为负数） |

### 同一 format+type 多个 sink

同一 `format` + `type` 组合可以配置多个 sink 条目，各 sink 独立运行，输出到不同的表或文件。

### 部分列覆盖（mergeDefaults）

diff-record 类型支持 `mergeDefaults: true`，保留所有默认输出列，仅对 `columns` 中匹配名称的列进行值替换，不匹配的列继续使用默认值。CSV/JSON 格式还会将 `columns` 中不属于默认列的额外列追加到末尾。

```yaml
result:
  sinks:
    # 默认宽表，仅覆盖 nl_dq_execution_id 的值来源
    - format: table
      type: diff-record
      properties:
        url: jdbc:mysql://localhost:3306/db
        username: root
        password: secret
        tableName: diff_detail
        mergeDefaults: true
        columns:
          - name: nl_dq_execution_id
            value: ${attr.customId}   # 覆盖默认 taskId 来源
    # CSV：默认 4 列 + 覆盖 operation + 追加自定义列
    - format: csv
      type: diff-record
      properties:
        path: ./output/diff-${taskId}.csv
        mergeDefaults: true
        columns:
          - name: operation
            value: "DIFF_${operation}"   # 覆盖
          - name: env
            value: production             # 追加额外列
```

默认列名（Table sink）：`nl_dq_execution_id`、`nl_dq_diff_type`、`nl_dq_diff_columns1`、`nl_dq_diff_columns2`、`{col}_1`、`{col}_2`

默认列名（CSV sink）：`operation`、`primaryKey`、`sourceValues`、`targetValues`

默认字段名（JSON sink）：`operation`、`primaryKey`、`sourceValues`、`targetValues`、`changedColumns1`、`changedColumns2`

```yaml
result:
  sinks:
    # 写入明细表（自定义字段）
    - format: table
      type: diff-record
      properties:
        url: jdbc:mysql://localhost:3306/db
        username: root
        password: secret
        tableName: diff_detail
        columns:
          - name: task_id
            value: ${taskId}
          - name: op
            value: ${operation}
          - name: pk
            value: ${primaryKey}

    # 再写入审计表（不同字段、不同表）
    - format: table
      type: diff-record
      properties:
        url: jdbc:mysql://localhost:3306/audit
        username: auditor
        password: secret
        tableName: audit_diff_log
        columns:
          - name: execution_id
            value: ${taskId}
          - name: src_tbl
            value: ${sourceTable}
          - name: tgt_tbl
            value: ${targetTable}
          - name: change_type
            value: ${operation}
```

## 六、标准化配置

标准化配置对应：

- [NormalizationConfig.java](../consilens-cli/src/main/java/com/consilens/cli/model/normalization/NormalizationConfig.java)
- [TypeNormalizationRule.java](../consilens-cli/src/main/java/com/consilens/cli/model/normalization/TypeNormalizationRule.java)

支持层级：

- `global`
- `source`
- `target`

优先级：

- 数据库级覆盖全局配置

示例：

```yaml
normalization:
  global:
    decimal:
      precision: 2
      rounding: true
    timestamp:
      format: yyyy-MM-dd HH:mm:ss
      timezone: UTC
  source:
    binary:
      encoding: hex
      uppercase: false
  target:
    boolean:
      trueValue: "1"
      falseValue: "0"
      nullValue: ""
```

当前版本说明：

- 支持按数据类型配置
- 不支持按列级单独配置

## 七、终局比较说明

默认无需额外配置，终局小段会使用 `full` 模式直接拉完整行比较。

如果确认当前场景适合先用行哈希过滤，可以显式配置：

```yaml
strategy:
  localCompare:
    mode: row-hash
```

## 九、常见误区

### 误区 1：`join` 可以跨库

不可以。`join` 当前会校验两侧 JDBC URL 完全一致。

### 误区 2：`local` 已可用

不可以。当前是预留值，配置校验阶段会直接拒绝。

### 误区 3：使用旧 `output` 节点配置输出

旧的 `output` 节点已从代码中移除，不再支持。请使用 `result.sinks[]` 配置输出。

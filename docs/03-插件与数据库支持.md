# 连接器与数据源支持

Consilens 的数据源适配通过连接器（Connector）插件实现，核心扩展接口包括：

- [ConnectorProvider.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/ConnectorProvider.java)
- [DatasetHandle.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/dataset/DatasetHandle.java)
- [DatabaseDialect.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/DatabaseDialect.java)
- [DatabaseDialectProvider.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/DatabaseDialectProvider.java)

## 一、当前仓库内置连接器

当前仓库实际提供的连接器模块有：

| 数据库 | 模块 |
| --- | --- |
| MySQL | `consilens-connector-mysql` |
| PostgreSQL | `consilens-connector-postgresql` |
| SQL Server | `consilens-connector-sqlserver` |
| Oracle | `consilens-connector-oracle` |
| Presto | `consilens-connector-presto` |
| Trino | `consilens-connector-trino` |
| Apache Doris | `consilens-connector-doris` |
| StarRocks | `consilens-connector-starrocks` |
| ClickHouse | `consilens-connector-clickhouse` |
| TiDB | `consilens-connector-tidb` |

说明：

- [DatabaseType.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/enums/DatabaseType.java) 中定义了更多枚举值
- 但"能否真正使用"取决于是否存在对应连接器模块和 SPI 注册
- 当前已完成端到端验证的数据库是 MySQL、PostgreSQL 和 StarRocks；其余内置连接器建议在真实场景接入前先自行验证

## 二、连接器加载机制

加载路径：

1. CLI 读取配置中的 `source.type` / `target.type` 和 `connection` 参数
2. `ConnectorConfigMapper` 将 CLI 配置映射为 `ConnectorConfig`
3. `ConnectorProvider` 根据连接器类型创建 `DatasetHandle`
4. 对于 JDBC 类连接器，`DialectFactory` 通过 `PluginManager + ServiceLoader` 加载 `DatabaseDialectProvider`
5. `DatasetHandle` 封装连接池与方言，向上暴露统一的数据集访问和能力查询

核心类：

- [ConnectorProvider.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/ConnectorProvider.java)
- [DatasetHandle.java](../consilens-connector/consilens-connector-api/src/main/java/com/consilens/connector/api/dataset/DatasetHandle.java)
- [DialectFactory.java](../consilens-core/src/main/java/com/consilens/core/database/dialect/DialectFactory.java)
- [PluginManager.java](../consilens-spi/src/main/java/com/consilens/plugin/runtime/PluginManager.java)

## 三、SPI 注册方式

以 MySQL 为例，插件声明文件位于：

- [META-INF/services/com.consilens.connector.api.DatabaseDialectProvider](../consilens-connector/consilens-connector-plugins/consilens-connector-mysql/src/main/resources/META-INF/services/com.consilens.connector.api.DatabaseDialectProvider)

内容示例：

```text
com.consilens.connector.mysql.MySQLDatabaseDialectProvider
```

这意味着：

- 运行时会通过 `ServiceLoader` 发现 `MySQLDatabaseDialectProvider`
- Provider 负责创建 `MySQLDatabaseDialect`

## 四、数据库方言职责

一个 `DatabaseDialect` 不只是"生成 SQL"，它聚合了多个子组件：

- `SqlQueryGenerator`
- `MetadataQueryGenerator`
- `DataTypeHandler`
- `TransactionManager`
- `CapabilityProvider`
- `ConnectionPoolOptimizer`

这套设计把数据库差异拆到明确职责中，而不是堆在一个巨大方言类里。

## 五、连接器在发行包中的位置

打包后，连接器插件和 JDBC 驱动会进入：

- `plugins/`

核心 CLI 与公共依赖位于：

- `libs/`

对应组装规则见：

- [assembly.xml](../consilens-dist/src/main/assembly/assembly.xml)

这意味着：

- 新增连接器后，只要模块被纳入构建并打包，就能通过 `plugins/` 加载
- 自定义部署时也可以通过补充 jar 的方式扩展

## 六、推荐的连接器类型配置

虽然多数场景能从 JDBC URL 自动识别类型，但建议在配置中显式写 `source.type` / `target.type`，这样更稳定。

示例：

```yaml
source:
  type: mysql
target:
  type: postgresql
```

常用取值：

- `mysql`
- `postgresql`
- `sqlserver`
- `oracle`
- `presto`
- `trino`
- `doris`
- `starrocks`
- `clickhouse`
- `tidb`

## 七、不同策略对连接器的要求

### `checksum`

- 支持跨数据源
- 分段阶段强依赖连接器提供 checksum SQL 生成能力（通过 `CapabilitySet` 声明 `SERVER_SIDE_HASH` 能力）
- 若显式配置 `strategy.localCompare.mode: row-hash`，还需要连接器提供 row-hash SQL 生成能力

### `join`

- 只支持同一个 JDBC URL
- 更依赖数据库 SQL JOIN 能力（通过 `CapabilitySet` 声明 `SERVER_SIDE_JOIN` 能力）

### `local`

- 当前未实现，配置校验阶段会直接拒绝

## 八、如何新增一个连接器

最小步骤：

1. 在 `consilens-connector-plugins` 下新建模块
2. 实现一个 `DatabaseDialect`
3. 提供对应的 SQL、元数据、类型处理、能力探测组件
4. 实现 `DatabaseDialectProvider`
5. 在 `META-INF/services/com.consilens.connector.api.DatabaseDialectProvider` 中注册
6. 将模块纳入父 POM 与发行包

实践上建议：

- 继承 [AbstractDatabaseDialect.java](../consilens-connector/consilens-connector-plugins/consilens-connector-base/src/main/java/com/consilens/conncetor/base/AbstractDatabaseDialect.java)
- 参考 [MySQLDatabaseDialect.java](../consilens-connector/consilens-connector-plugins/consilens-connector-mysql/src/main/java/com/consilens/connector/mysql/MySQLDatabaseDialect.java) 的组织方式

### 扩展到非 JDBC 数据源

当前架构已预留了非 JDBC 数据源的扩展能力。连接器只需要：

1. 实现 `ConnectorProvider`，在 `createDatasetHandle()` 中返回自定义 `DatasetHandle`
2. `DatasetHandle` 通过 `getCapabilities()` 声明支持的能力（如 `SERVER_SIDE_HASH`）
3. 核心的比对规划器 `DefaultComparePlanner` 仅基于能力集合选择执行计划，不区分数据源类型

这意味着未来的 Elasticsearch、MongoDB、HDFS 等连接器，只要实现对应的 `DatasetHandle` 并声明足够的能力，就可以复用现有的比对流程。

## 九、连接器开发注意事项

- 保证 checksum 语义跨数据源可比
- 保证 row-hash 语义与 checksum 阶段使用的标准化规则一致
- 类型标准化要与目标数据源保持一致
- 元数据查询要兼容大小写、schema、catalog 差异
- 连接参数通过 `connection` 配置块暴露给用户

## 十、常见问题

### 为什么枚举里有某数据库，但我不能直接用？

因为枚举只是类型定义，真正可用性取决于：

- 是否有连接器模块
- 是否有 Provider 注册
- 是否把该连接器打进发行包或部署目录

### 可以把连接器单独部署到 `plugins/` 吗？

可以，只要满足：

- jar 内包含 SPI 注册文件
- SPI 注册文件为 `META-INF/services/com.consilens.connector.api.DatabaseDialectProvider`
- 依赖满足运行时类路径要求

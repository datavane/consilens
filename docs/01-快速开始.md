# 快速开始

## 适用对象

本文适合第一次使用 Consilens 的用户，目标是在最短时间内完成一次跨数据库数据比对。

## 环境要求

- JDK 11 及以上
- 可访问源库和目标库的 JDBC 网络
- 两侧表都能通过主键唯一定位
- 建议预留 8G JVM 堆内存

推荐 JVM 参数：

```bash
-Xmx8g -Xms8g
```

## 构建

在仓库根目录执行：

```bash
mvn clean package -DskipTests -Prelease
```

产物位置：

- consilens-dist/target/consilens-${version}.tar.gz

## 解压运行

```bash
tar -xzf consilens-dist/target/consilens-${version}.tar.gz
cd consilens-${version}
./bin/consilens-cli.sh --help
```

发行包结构：

- `bin/`：启动脚本与环境变量脚本
- `conf/`：日志配置
- `libs/`：CLI 与核心依赖
- `plugins/`：数据库连接器与 JDBC 驱动
- `examples/`：打包时拷入的示例配置
- `logs/`：运行日志

## 最小可用配置

```yaml
source:
  type: mysql
  url: jdbc:mysql://localhost:3306/source_db
  username: source_user
  password: change_me_source_password

target:
  type: postgresql
  url: jdbc:postgresql://localhost:5432/target_db?currentSchema=public
  username: target_user
  password: change_me_target_password

comparison:
  tables:
    source: orders
    target: orders
  keys:
    source:
      - order_id
    target:
      - order_id
  compareColumns:
    source:
      - order_id
      - customer_id
      - amount
      - status
      - created_at
    target:
      - order_id
      - customer_id
      - amount
      - status
      - created_at

strategy:
  mode: checksum
  algorithm: xor
  bisectionFactor: 8
  bisectionThreshold: 5000
  batchSize: 1000
  enableProfiling: false

result:
  sinks:
    - format: console
      type: result
    - format: json
      type: diff-record
      properties:
        path: ./diff_results.json
        pretty: true
```

说明：

- `comparison.keys.source` 和 `comparison.keys.target` 必须一一对应，数量必须一致。
- `comparison.compareColumns.source` 和 `comparison.compareColumns.target` 是实际参与数据比较的列，通常应包含主键列。
- `source.type` / `target.type` 可省略；省略时根据 JDBC URL 自动识别。

## 执行比对

```bash
./bin/consilens-cli.sh config validate -c config.yaml
./bin/consilens-cli.sh diff -c config.yaml
```

请先把 `config.yaml` 中的用户名和密码替换为真实值。配置文件支持环境变量占位符，语法为 `${env.VAR_NAME}` 或 `${env.VAR_NAME:默认值}`，例如 `password: ${env.DB_PASSWORD:change_me}`。

## 常用命令

### 查看帮助

```bash
./bin/consilens-cli.sh --help
```

### 查看版本

```bash
./bin/consilens-cli.sh --version
```

### 生成模板

```bash
./bin/consilens-cli.sh config generate -o config.yaml
./bin/consilens-cli.sh config generate -o advanced.yaml -t advanced
```

### 校验配置

```bash
./bin/consilens-cli.sh config validate -c config.yaml
```

## 结果解读

常见差异类型：

- `SOURCE_MISSING`：目标表存在，源表不存在
- `TARGET_MISSING`：源表存在，目标表不存在
- `MISMATCH`：主键相同，但非主键字段不一致

当开启 `infoTree` 输出时，可以看到每一段的：

- 行数
- 分裂方式
- checksum 是否一致
- 最终落到本地比较的段

## 使用建议

### 首次接入

- 先用一张小表验证配置正确性。
- 再放大到目标表。

### 大表

- 优先使用 `strategy=checksum`
- `algorithm` 推荐 `xor`
- 终局本地比较会自动在完整行比较和 row-hash 过滤之间选择

### 同库同实例

- 可考虑 `strategy=join`
- `join` 仅支持两张表位于同一个 JDBC URL

### 需要落库或写文件

- 在 `result.sinks[]` 中配置 `format: table` 类型 sink，并填写数据库连接参数
- 也可配置 `format: json` 或 `format: csv` 写出到文件
